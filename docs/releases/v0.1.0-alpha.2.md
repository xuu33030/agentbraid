# AgentBraid v0.1.0-alpha.2

This corrective alpha release restores Codex structured-output compatibility and hardens run
delivery, cancellation, concurrency, and capability routing. The Python package version is
`0.1.0a2`; the intended Git and GitHub release tag is `v0.1.0-alpha.2`.

## Highlights

- Normalize Codex output schemas for strict structured-output validation.
- Bind every run to its original branch and commit before explicit local apply.
- Reject dirty shared worktrees after non-mutating host tasks.
- Cancel active Codex subprocesses when a run is cancelled.
- Serialize final review and honor configured Codex task parallelism.
- Apply the global retry ceiling and isolate capability history by exact model.
- Persist planning, task, and review token usage in SQLite schema v2.

## Upgrade notes

- Existing schema v1 databases migrate automatically to schema v2 on first startup.
- Historical runs created before v2 have no recorded delivery target and cannot be applied.
- Existing `v0.1.0-alpha.1` installations should upgrade because that release predates the Codex
  output-schema compatibility fix.

Publication, checksums, and the final verification record remain separate release actions after
the corrective pull request is merged.
