# AgentBraid v0.2.0-alpha.3

This alpha adds a fixed Dashboard guide and quality-first model selection while preserving the
local provider boundary. The Python package version is `0.2.0a3`; the intended Git and GitHub
release tag is `v0.2.0-alpha.3`.

## Highlights

- Open the sticky **Guide** on desktop or mobile for safely quoted macOS/Linux and PowerShell
  setup, daily-start, model, Dashboard, and AGY TUI commands.
- Refresh Codex choices through `codex debug models` with bundled fallback, and AGY choices through
  the fixed `agy models` command. Editable model inputs remain available when either catalog fails.
- Choose quick, standard, complex, or high-risk work and review the top three quality-first Codex
  and AGY recommendations before explicitly applying them to a form.
- Set Codex reasoning effort per workspace or run without changing `config.toml`. Unsupported
  automatic choices fall back to the closest supported level not above the target; `max` and
  `ultra` are never recommended automatically.
- Inspect benchmark sources, raw evidence, update metadata, licensing status, and local efficiency
  tie-break details. Restricted sources remain reference-only and do not influence scoring.

## Security and privacy

- Catalog commands use fixed argument vectors, no shell, a 10-second timeout, and bounded output.
  AgentBraid does not start the AGY TUI, edit AGY settings, or read provider credentials.
- External score updates are opt-in and use only the fixed AgentBraid GitHub manifest URL. The
  Dashboard sends no workspace path, goal, usage, token, or provider data to that endpoint.
- POSIX and PowerShell commands are quoted on the server. The browser only copies text and never
  opens or controls an operating-system terminal.

## Upgrade notes

- Existing schema v1 through v4 databases migrate sequentially to schema v5 on first startup.
  Existing provider usage remains readable with a `NULL` reasoning effort.
- Existing run snapshots and Pydantic defaults remain compatible. New runs persist the resolved
  Codex model and reasoning effort so later workspace-default changes cannot alter them.
- A selected AGY model updates routing metadata and generated startup commands only. Start a new
  AGY session with that command or choose `/model` inside AGY to change an active session.

Publication, checksums, and the final verification record remain separate release actions after
the feature pull request is merged.
