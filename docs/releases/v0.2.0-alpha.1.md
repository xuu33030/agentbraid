# AgentBraid v0.2.0-alpha.1

This alpha introduces AgentBraid's first local visual control surface. The Python package version
is `0.2.0a1`; the intended Git and GitHub release tag is `v0.2.0-alpha.1`.

## Highlights

- Launch an authenticated local Web App with `agentbraid dashboard .`.
- Browse the current workspace or every workspace recorded in the active state database.
- Inspect task dependencies, executor assignments, retries, durable events, capability health,
  final review, and Git delivery readiness.
- Visualize observed Codex input, cached input, output, reasoning output, duration, attempts, and
  objective retry overhead without double-counting token subsets.
- Cancel an active run from a separate Dashboard process and propagate cancellation to planning,
  worker, or review Codex subprocesses.
- Apply a reviewed integration branch only after the original branch, commit, and clean-workspace
  preflight passes and the user types `apply-reviewed-run`.

## Security model

- The Dashboard binds only to `127.0.0.1`; remote binding is not configurable.
- A single-use bootstrap token establishes an HttpOnly, SameSite-strict session for the life of
  the Dashboard process.
- State-changing requests require the expected Host, same Origin, session cookie, and CSRF token.
- HTML, CSS, and JavaScript are bundled in the wheel. No CDN, remote script, filesystem browser,
  provider credential access, automatic push, or deployment path is added.

## Upgrade notes

- Existing schema v1 and v2 databases migrate sequentially to schema v3 on first startup.
- Historical v2 provider usage remains visible but is labelled legacy because it has no persisted
  attempt or outcome attribution.
- "All projects" means all workspaces represented in the active state database. AgentBraid does
  not scan the filesystem or discover unrelated custom databases.
- The Dashboard reports observed Codex usage only. It does not estimate Antigravity usage,
  subscription quota remaining, or monetary cost.
- Starting runs and completing host tasks remain in Antigravity; the Dashboard does not replace
  the official authenticated host.

Publication, checksums, and the final release record remain separate actions after the feature
pull request is merged.
