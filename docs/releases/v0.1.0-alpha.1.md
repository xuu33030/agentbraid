# AgentBraid v0.1.0-alpha.1

AgentBraid's first public alpha braids an accountable Codex lead with tasks executed by Codex or
an official Antigravity MCP host. The Python package version is `0.1.0a1`; the Git and GitHub
release tag is `v0.1.0-alpha.1`.

> Alpha warning: MCP schemas and persisted SQLite state may change incompatibly before 1.0. Use a
> test repository first and keep normal source-control backups.

## Highlights

- Codex creates a typed task DAG, supplies routing preferences, executes assigned tasks, and
  resumes the same lead thread for final review.
- Antigravity loads AgentBraid through the officially supported workspace MCP and Skill paths,
  claims only host-assigned tasks, and submits typed evidence.
- Deterministic routing records why each task selected Codex or the active host.
- Mutating work runs in per-task Git worktrees and reaches a local integration branch only after
  validation, commit verification, dependency ordering, and conflict-safe cherry-pick.
- Delivery requires a completed Codex review plus a separate `apply_run` call with the literal
  confirmation `apply-reviewed-run`.
- SQLite state survives MCP process restarts while common credential patterns are redacted before
  persistence.

## Provider boundary

AgentBraid never launches `agy`, reads Antigravity or Google credential storage, proxies an
Antigravity login, or shares a user's subscription. Antigravity remains the official
user-authenticated host and invokes AgentBraid as a local MCP tool.

Codex runs through the documented `codex exec` interface using the user's existing Codex CLI
login. AgentBraid removes API-key-like environment variables from child processes and does not
store Codex credentials.

See the release-reviewed [`provider-policy.md`](../provider-policy.md) and
[`security-boundaries.md`](../security-boundaries.md).

## Install

Download `agentbraid-0.1.0a1-py3-none-any.whl` from this release, then install it into a virtual
environment you will keep:

```bash
python -m venv ~/.venvs/agentbraid
source ~/.venvs/agentbraid/bin/activate
python -m pip install ./agentbraid-0.1.0a1-py3-none-any.whl
agentbraid --version
```

From a clean target Git repository:

```bash
agentbraid doctor .
agentbraid init .
agy
```

Use `/mcp` and `/skills` to verify discovery, then invoke `/agentbraid` with a bounded goal. The
full walkthrough is in [`../getting-started.md`](../getting-started.md).

## Known limitations

- This release supports local stdio MCP only; there is no hosted multi-tenant service.
- Codex tasks execute sequentially, even though the task graph can expose independent work.
- Antigravity is the only tested user-facing host in v0.1.
- A host must create a local DCO-signed commit for successful mutating tasks.
- Integration conflicts preserve the failed worktree and block the run; automatic conflict repair
  is not included.
- AgentBraid does not push, deploy, publish packages, or mutate production systems.
- Runtime state has no stable migration guarantee during alpha.

## Checksums

The release includes `SHA256SUMS` for the wheel and source archive. Verify before installation:

```bash
shasum -a 256 -c SHA256SUMS
```

On Windows PowerShell, compare `Get-FileHash -Algorithm SHA256` output with the same file.
