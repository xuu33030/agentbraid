# AgentBraid v0.2.0-alpha.2

This alpha turns the local Dashboard into a run-management surface while preserving AgentBraid's
provider and delivery boundaries. The Python package version is `0.2.0a2`; the intended Git and
GitHub release tag is `v0.2.0-alpha.2`.

## Highlights

- Start runs from the authenticated Dashboard with actual Codex model selection and per-run
  routing, delivery, workspace-access, concurrency, retry, timeout, and output settings.
- Save workspace defaults while keeping database and state paths read-only. Codex executable and
  worktree-directory changes require a Dashboard and MCP restart.
- Generate and edit concise English, Traditional Chinese, and Simplified Chinese run names without
  modifying the original goal.
- Select run records and delete their SQLite state together with safely removable AgentBraid
  worktrees and branches after typing `delete-selected-runs`.
- Preserve dirty worktrees, unmerged integration branches, task branches with unique patches,
  active runs, unavailable workspaces, and every corresponding database record.
- Keep AGY model selection as routing metadata only. The Dashboard never launches, controls, or
  authenticates as Antigravity.

## Upgrade notes

- Existing schema v1, v2, and v3 databases migrate sequentially to schema v4 on first startup.
- Existing runs remain readable. Runs created before schema v4 have no immutable execution
  snapshot or localized names and fall back to their original goal and current compatible defaults.
- Runtime settings are resolved and persisted when a run is created, so later workspace-default
  changes do not alter that run.
- Dashboard deletion is intentionally conservative and never force-removes user work. Resolve a
  displayed blocker or preserve the run for manual recovery.
- Hybrid runs may still wait for the authenticated Antigravity MCP host to claim host-assigned
  tasks. Choose Codex-only routing when no host task should be created.

Publication, checksums, and the final verification record remain separate release actions after
the feature pull request is merged.
